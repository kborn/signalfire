Long autonomous agent sessions against an unclear or shifting target don't work for this project. The user tried "throw one session at remaining work" (Phase 13) and it produced lateral movement and regressions.

**What works:**

- Small scoped subphases with a defined exit condition
- User reviews output before the next subphase begins
- Working from a prioritized fix list derived from review findings, not from the previous session's deferred items
- Agent leads design decisions and defends them — user preference interjections on visual details (image in/out, motif on/off, logo iterations) cause churn

**What doesn't work:**

- Multi-hour autonomous runs without check-ins
- User deferring all visual decisions to the agent while also overriding them mid-session
- Working on Phase N.x polish items when review findings reveal Phase N+1 structural problems
- Asking the agent to "make it look better" without specifying which review finding to fix

**Why:** The user acknowledged they keep "throwing preferences" and worrying they're preventing progress. They're not the main blocker — the bigger issue is working from the wrong list. Fix the product logic issues before visual polish.

**How to apply:** At the start of every session, agree on which specific fix list items are in scope. No scope expansion mid-session unless blocked. User reviews after each subphase, not after the full phase.
