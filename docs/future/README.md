# Future Planning Notes

`docs/future/` holds non-canonical planning notes for approved future work that
is not yet implementation-ready enough to live in canonical specs.

Use this directory for:

- raw but durable milestone or roadmap ideas
- strategy notes that still contain open questions
- review notes that should inform future phase planning

Do not use this directory for:

- current project truth
- active implementation contracts
- stale historical material that is no longer relevant

Canonical authority remains in:

- `docs/agent-governance/`
- `docs/specs/`
- `docs/architecture/`

Agent expectation:

- consult `docs/future/` when refining future phase tasks or roadmap planning
- promote durable decisions into canonical docs when they become approved
- tidy or remove future notes after they are implemented or superseded
