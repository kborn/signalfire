# Milestone 1 UI Polish Review Notes

## Purpose

Preserve the external-review UI/UX findings that should inform the final
Milestone 1 polish pass without treating them as an active implementation spec.

Use `docs/agent-governance/progress.md` as the canonical owner of the actual
Phase 15.5 polish tasks.

---

## Key Review Themes

- strengthen card definition on dark backgrounds
- make primary CTAs read clearly as primary
- contain the homepage hero illustration so it does not bleed into later
  sections unintentionally
- create a clearer separation between article metadata and article body content
- format event dates/times for humans rather than exposing UTC-only or raw ISO
  strings
- visually differentiate topic cards more effectively
- improve the homepage section rhythm so the scroll experience feels more
  intentional
- keep the hero focused on exploration rather than mission-statement copy

---

## Additional Editorial Note

The review also suggested a moderation/publishing contract for article Markdown:

- no body-level H1 because the page already renders the title from metadata
- begin with a lead paragraph before H2 sections
- use H2 for major sections and H3 sparingly
- prefer lists for scannable actions/resources
- end with a concrete action-oriented section

This is a useful editorial heuristic, but it should only become canonical if it
is later promoted into a moderation or content-style document deliberately.
