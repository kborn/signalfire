# Implementation Guides

Store phase-scoped framework guides in this directory.

An implementation guide is different from a syllabus:

- a syllabus explains what to learn and in what order
- an implementation guide explains how the relevant framework concepts work for the current task

Use this directory when the human is blocked on framework mechanics for the
current phase, for example:

- what `page.tsx` and `layout.tsx` mean in Next.js
- how a dynamic route like `[slug]` works
- what a component is
- how props, JSX, or TSX syntax map to the task at hand
- how a framework concept shows up in this repo's current phase

Recommended contents:

1. Concepts to understand now
2. Plain-language explanations
3. Minimal worked examples
4. How the concept appears in this repo's code
5. Tiny rules of thumb

These guides are concept-teaching artifacts. They should not try to own the
step-by-step repo build order. If the human needs file order or edit order,
create a walkthrough in `docs/learnings/walkthroughs/` instead.

These guides are committed repo artifacts, so they must reference only
committed repo files, not `.ai/` paths.
