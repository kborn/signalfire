# Learning Workflow User Guide

Use this guide when you want AI help that teaches, not just answers.

## What To Ask For

### Ask for a syllabus when you need task orientation

Use a syllabus when you want help understanding:

- what this phase or subtask is trying to accomplish
- what is in scope right now
- what concepts matter first

Example prompts:

- `I'm starting Phase 6.1. Generate a syllabus.`
- `Give me a syllabus for this subtask so I understand the scope before coding.`

### Ask for an implementation guide when you need framework teaching

Use an implementation guide when you want help understanding:

- what a framework concept means
- how that concept works in the current phase
- tiny examples in plain language

Example prompts:

- `Create an implementation guide for Phase 6.1 Next.js routing and layout concepts.`
- `I don't understand page.tsx, layout.tsx, or [slug]. Make an implementation guide for this phase.`

### Ask for a walkthrough when you need repo steps

Use a walkthrough when you want help understanding:

- what files or folders to touch
- what order to work in
- what the first correct structure should look like in this repo

Example prompts:

- `Create a walkthrough for Phase 6.1 route setup in this repo.`
- `I understand the concepts now. Show me what files to create first.`

### Ask for index cards when you want short reminders

Use index cards when you want compact review material for concepts that keep
coming up.

Example prompts:

- `Make index cards for the Phase 6.1 Next.js concepts I keep getting stuck on.`
- `Add a card for the difference between layout.tsx and page.tsx.`

## Quick Rule Of Thumb

- `What am I doing?` -> syllabus
- `What does this mean in Next.js/React/Node?` -> implementation guide
- `What do I edit first in this repo?` -> walkthrough
- `I keep forgetting this concept.` -> index card

## Ask-When-Stuck Prompts

If you are blocked, ask narrow questions like:

- `What does layout.tsx do differently from page.tsx?`
- `Why does [slug] create a dynamic route?`
- `Why is this file .tsx instead of .ts?`
- `When do I need 'use client'?`
- `What files should exist for Phase 6.1?`

## Suggested Workflow

For a new phase or subtask:

1. Ask for a syllabus.
2. Ask for an implementation guide for the framework concepts you do not know.
3. Ask for a walkthrough when you are ready to edit files.
4. Add index cards for anything you expect to forget.
