# Learning Card 09: Java -> Node Critical Path

If learning Node backend work, understand these first:

1. Event loop: blocking sync work can stall the app
2. Promises / `async` / `await`: async results arrive later
3. Nest wiring: modules wire providers; they do not hold business logic
4. TypeScript is not Java: types help, but runtime behavior is still JS
5. Prisma is generated from schema: editor autocomplete reflects your data model

If confused, ask:

- runtime problem?
- async problem?
- Nest DI problem?
- Prisma/schema problem?
