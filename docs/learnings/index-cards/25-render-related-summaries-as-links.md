# Render Related Summaries As Links

If an API already includes related content summaries with slugs, those summaries
should usually become clickable next-step navigation, not dead text.

Why it matters here:
Phase 6.2 topic detail already receives related article and action summaries, so
the page can link users directly into `/articles/[slug]` and `/actions/[slug]`.

Rule of thumb:
When a summary is part of a discovery flow, make the title a clear internal
link and keep the supporting summary text nearby.
